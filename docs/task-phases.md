# AI 面试抽题网站任务阶段划分

## 阶段 0：项目初始化

目标：搭好可开发、可运行、可连接基础服务的项目骨架。

主要任务：

- 初始化 Next.js App Router + TypeScript 项目。
- 接入 Tailwind CSS 和 shadcn/ui。
- 配置 ESLint、Prettier、基础 tsconfig。
- 接入 Prisma。
- 配置 PostgreSQL 连接。
- 配置 Redis 连接。
- 建立 `.env.example` 和 env 校验。
- 建立基础页面布局和导航。

关键产出：

- 本地开发服务可启动。
- 数据库连接可用。
- Redis 连接可用。
- 基础页面可访问。

验收标准：

- `npm run dev` 可以启动应用。
- `npm run lint` 通过。
- Prisma 可以连接目标 PostgreSQL。
- Redis ping 成功。

## 阶段 1：账号与基础后台

目标：完成内部使用所需的基础登录和后台框架。

主要任务：

- 实现简单邮箱密码登录。
- 创建 `users` 表。
- 实现登录页、退出、会话保护。
- 实现工作台首页。
- 实现统一页面布局、侧边栏和顶部栏。

关键产出：

- 登录后才能访问后台页面。
- 工作台能展示基础占位数据。

验收标准：

- 未登录访问后台会跳转登录页。
- 登录成功后进入工作台。
- 退出后会话失效。

## 阶段 2：题库与题目管理

目标：先把核心数据模型和手动维护能力做出来。

主要任务：

- 设计并迁移核心表：
  - `question_banks`
  - `questions`
  - `tags`
  - `question_tags`
- 实现题库 CRUD。
- 实现题目 CRUD。
- 支持题目标签和难度分数。
- 支持按题库、标签、难度、关键词筛选。
- 支持启用/禁用题目。

关键产出：

- 可以手动创建题库。
- 可以手动添加题目、答案、标签和难度分数。
- 可以搜索和筛选题目。

验收标准：

- 创建题库后能在列表中看到。
- 新增题目后能关联到题库。
- 题目必须保存 `difficulty_score` 和标签。
- 禁用题目不会进入后续抽题候选集。

## 阶段 3：飞书 API 导入

目标：打通飞书文档到系统导入预览的链路。

主要任务：

- 实现 Feishu Client。
- 支持获取并缓存 `tenant_access_token`。
- 实现飞书文档链接解析。
- 实现文档 blocks 分页拉取。
- 实现 block 到 normalized text 的转换。
- 创建导入相关表：
  - `feishu_sources`
  - `import_batches`
  - `import_items`
- 实现飞书导入页面。
- 实现导入任务状态展示。

关键产出：

- 输入飞书文档链接后，可以拉取文档内容。
- 系统保存原始 blocks 和归一化文本。
- 导入批次状态可追踪。

验收标准：

- 有权限的飞书文档可以成功读取。
- 无权限、链接错误、限流等场景有明确错误提示。
- 同一导入批次能追溯飞书来源 URL 和 document ID。

## 阶段 4：AI 抽取题目与导入确认

目标：让 AI 从飞书内容中抽题，并由用户确认后入正式题库。

主要任务：

- 实现统一 LLM Client。
- 配置 `deepseek4flash`。
- 默认关闭思考模式。
- 编写飞书内容抽题 prompt。
- 定义 AI 抽题 JSON Schema。
- 实现 AI 输出校验。
- 将 AI 抽取结果写入 `import_items`。
- 实现导入确认页面。
- 支持编辑问题、答案、标签、难度分数。
- 支持批量确认入库。

关键产出：

- 飞书文档内容可以被 AI 转成结构化题目。
- 用户能在入库前检查和修改。
- 确认后题目进入正式题库。

验收标准：

- 每道入库题都有问题、标签、难度分数和来源信息。
- AI 低置信度结果有提示。
- 批量入库使用事务，失败时整体回滚。
- AI JSON 解析失败时有重试或失败提示。

## 阶段 5：大模型智能抽题

目标：实现由大模型按练习目标智能选择题目，而不是简单随机。

主要任务：

- 创建练习相关表：
  - `practice_sessions`
  - `practice_session_questions`
- 实现练习配置页。
- 支持选择题库、题量、标签、难度倾向和练习目标。
- 实现候选题筛选。
- 将候选题摘要和历史表现传给大模型。
- 实现大模型抽题 prompt。
- 校验大模型返回的题目 ID。
- 保存抽题理由和整体策略。
- 实现规则兜底抽题。

关键产出：

- 用户可以开始一轮练习。
- 本轮题目由大模型选择。
- 每题保存抽题理由。

验收标准：

- 抽题结果只包含启用且符合筛选条件的题目。
- 每道被抽中的题都有 reason。
- 大模型失败时系统仍能生成练习会话。
- 最近练过的题可以被降权。

## 阶段 6：AI 面试练习与总结

目标：完成一次从抽题、回答、评分到总结的完整闭环。

主要任务：

- 实现练习页面。
- 支持逐题回答。
- 创建 `practice_answers` 表。
- 调用 AI 对回答评分。
- 输出亮点、缺失点、参考表达和追问。
- 支持进入下一题。
- 实现会话总结。
- 统计薄弱标签和推荐复习题。

关键产出：

- 用户可以完成一轮 AI 面试练习。
- 每题回答和 AI 反馈被保存。
- 会话结束后生成总结。

验收标准：

- 用户回答后能得到 0 到 100 分评分。
- 单题反馈包含缺失点和改进表达。
- 会话总结包含总体评分、薄弱标签和下一轮建议。
- 历史练习记录可被后续抽题使用。

## 阶段 7：上线前整理

目标：完成 MVP 上线前的稳定性和可维护性检查。

主要任务：

- 补充关键错误日志。
- 隐藏敏感信息。
- 增加加载态、空状态和错误状态。
- 增加基础种子数据。
- 编写部署说明。
- 编写飞书应用配置说明。
- 完成一次端到端回归测试。

关键产出：

- MVP 可以部署给内部用户使用。
- 新用户知道如何配置飞书和环境变量。

验收标准：

- 全链路可跑通：登录 -> 飞书导入 -> AI 抽题入库 -> 大模型抽题 -> 回答评分 -> 总结。
- `.env` 不会被提交。
- 错误信息对用户可理解。
- README 或部署文档可指导启动项目。

## 推荐执行顺序

```text
阶段 0 项目初始化
-> 阶段 1 账号与基础后台
-> 阶段 2 题库与题目管理
-> 阶段 3 飞书 API 导入
-> 阶段 4 AI 抽取题目与导入确认
-> 阶段 5 大模型智能抽题
-> 阶段 6 AI 面试练习与总结
-> 阶段 7 上线前整理
```

## 最小可用闭环

如果想最快看到效果，可以把第一轮目标压缩为：

1. 阶段 0：项目能运行。
2. 阶段 2：能手动建题库和题目。
3. 阶段 3：能从飞书拉内容。
4. 阶段 4：AI 能抽题并确认入库。
5. 阶段 5：大模型能抽出一组题。

完成这五步后，即使 AI 评分和总结还没做，也已经具备核心价值。
