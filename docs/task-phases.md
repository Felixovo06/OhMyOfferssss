# AI 面试抽题网站任务书

## 执行方式

项目按前后端分离推进：

- 前端：TypeScript / Next.js，负责 UI、交互、状态和用户体验。
- 后端：Python / FastAPI，负责 API、数据、权限、飞书、AI 和任务编排。

任务阶段保持少量大阶段，适合 vibecoding 快速迭代。每个阶段都分成：

- 后端任务
- 前端任务
- 联调验收

后端优先提供 OpenAPI 契约和 mock 响应，前端可以基于 mock 先做 UI。涉及 UI 结构、视觉设计、交互体验、组件状态、响应式布局或 UI 质量评审时，必须使用 `ui-ux-pro-max`。

## 阶段 1：项目骨架、登录与小组

目标：前后端项目能独立启动，登录、小组和基础权限能打通。

### 后端任务

- 创建 `apps/api` Python FastAPI 项目。
- 配置 PostgreSQL、Redis、SQLAlchemy、Alembic。
- 建立基础 OpenAPI 文档。
- 实现用户登录、退出、当前用户接口。
- 实现小组创建、邀请、加入、成员列表接口。
- 实现基础鉴权和权限校验。

### 前端任务

- 创建 `apps/web` Next.js 项目。
- 建立 API client 基础封装。
- 实现登录页。
- 实现工作台首页。
- 实现小组创建、邀请、加入和成员列表页面。
- 使用 `ui-ux-pro-max` 设计/评审基础布局、登录页、工作台和小组流程。

### 联调验收

- 前端和后端都能本地启动。
- 前端能调用后端 `/api/v1/auth/me`。
- 用户能登录和退出。
- 用户能创建小组并邀请成员。
- OpenAPI 文档能看到基础接口。

## 阶段 2：题库与飞书导入

目标：用户能维护题库，并把飞书文档经过 AI 抽取后确认入库。

### 后端任务

- 实现题库 CRUD API。
- 实现题目 CRUD API。
- 支持标签、难度分数、个人题库和小组题库。
- 实现飞书 token 获取和 Redis 缓存。
- 实现飞书文档链接解析。
- 实现飞书 blocks 拉取。
- 实现 blocks 转 normalized text。
- 实现 AI 从飞书内容抽取题目、答案、标签、难度分数。
- 实现导入批次、导入项、导入确认和批量入库。

### 前端任务

- 实现题库列表。
- 实现题库详情。
- 实现题目新增、编辑、删除、启用/禁用。
- 实现标签、难度、关键词筛选。
- 实现飞书导入页。
- 实现导入状态展示。
- 实现导入确认页。
- 支持编辑待导入题、删除待导入题、批量确认入库。
- 使用 `ui-ux-pro-max` 设计/评审题库管理、导入流程、确认表格、低置信度提示和错误状态。

### 联调验收

- 用户能创建个人题库和小组题库。
- 用户能手动添加题目。
- 用户能输入飞书文档链接并拉取内容。
- AI 能生成待确认题目。
- 用户确认后题目入库。
- 每道题保存标签、难度分数和来源信息。

## 阶段 3：普通面试

目标：用户可以基于题库开始普通面试，大模型负责抽题，用户可以回答并获得反馈。

### 后端任务

- 实现候选题筛选。
- 实现普通面试抽题 prompt。
- 实现大模型抽题。
- 保存抽题理由和整体策略。
- 实现规则兜底抽题。
- 实现面试会话、面试题目、回答记录。
- 实现回答评分、追问和单题反馈。
- 实现会话总结。

### 前端任务

- 实现普通面试配置页。
- 支持选择题库、标签、难度、题量和目标。
- 展示大模型抽题结果和抽题理由。
- 实现面试答题页。
- 实现单题反馈、追问、进度和总结页。
- 使用 `ui-ux-pro-max` 设计/评审面试配置、答题体验、反馈面板、进度和总结页。

### 联调验收

- 用户能选择题库、标签、难度、题量和目标。
- 大模型能返回题目组合和抽题理由。
- 用户能逐题回答。
- AI 能给出评分、缺失点、参考表达和追问。
- 完成后能看到总结。

## 阶段 4：客制化面试

目标：用户上传 PDF 简历后，可以基于简历和题库进行客制化面试。

### 后端任务

- 实现 PDF 简历上传接口。
- 实现 PDF 文本解析。
- 判断扫描件或图片型 PDF。
- 实现简历结构化摘要。
- 实现客制化面试抽题 prompt。
- 大模型结合简历摘要和题库候选题抽题。
- 保存与简历相关的追问方向。
- 客制化面试回答评分时参考简历上下文。

### 前端任务

- 实现简历上传入口。
- 实现简历解析状态展示。
- 展示简历摘要、技能、经历、项目和追问方向。
- 实现扫描件 PDF 提示。
- 实现客制化面试配置页。
- 复用面试答题和总结体验。
- 使用 `ui-ux-pro-max` 设计/评审简历上传、解析状态、扫描件提示、简历摘要展示和客制化面试体验。

### 联调验收

- 用户能上传文本型 PDF 简历。
- 系统能解析简历文本并生成摘要。
- 扫描件 PDF 有明确提示。
- 大模型抽题理由能体现简历技能、项目或经历。
- AI 追问不能编造简历中不存在的经历。
- 用户能完成一轮客制化面试并看到总结。

## 阶段 5：联调、部署与打磨

目标：把 MVP 打磨到可以给内部用户使用。

### 后端任务

- 补齐错误日志和 requestId。
- 检查敏感信息。
- 完成环境变量说明。
- 完成飞书应用配置说明。
- 补充关键后端测试。
- 导出稳定 OpenAPI 文档。

### 前端任务

- 补齐加载态、空状态、错误态。
- 补齐危险操作二次确认。
- 做移动端基础可用性检查。
- 完成前端环境变量说明。
- 使用 `ui-ux-pro-max` 做上线前 UI/UX 评审。

### 联调验收

- 登录 -> 创建小组 -> 创建题库 -> 飞书导入 -> AI 抽题入库 -> 普通面试 -> 总结 全链路可跑通。
- 上传简历 -> 解析简历 -> 客制化面试 -> 总结 全链路可跑通。
- OpenAPI 契约与前端调用一致。
- 关键错误有可理解提示。
- `.env` 不会被提交。
- README 能指导本地启动和部署。

## 阶段 6：智能项目面试与知识库联动

目标：让面试从固定抽题升级为由大模型编排的真实面试流程。系统需要基于简历、岗位目标和题库语义，自动决定先问项目还是先问知识点，并在面试过程中根据候选人的回答动态追问、切换知识点和补充相关题目。

### 后端任务

- 扩展题库元信息，支持题库描述、适配岗位、技能关键词、领域标签和语义画像。
- 扩展简历解析结果，结构化保存项目经历、技术栈、职责范围、业务场景、亮点、风险点和可追问方向。
- 实现面试流程模式配置，支持“项目优先”和“知识优先”两种流程。
- 实现智能面试阶段状态，记录当前阶段、已覆盖项目、已覆盖知识点、已问题目和后续追问计划。
- 实现大模型题库匹配能力，根据简历摘要、岗位目标和题库元信息选择相关题库并给出选择理由。
- 实现大模型面试规划能力，生成本轮面试的阶段顺序、项目追问方向、知识点覆盖目标和题量分配。
- 实现项目追问 prompt，提供示例让大模型学会围绕项目背景、架构选择、技术细节、问题排查、性能优化和取舍原因追问。
- 实现回答后动态决策能力，根据用户回答判断继续追问项目、转入相关知识库题、补充随机相关题，或进入下一阶段。
- 实现知识库联动抽题能力，把项目中出现的技术点映射到候选题库和题目标签。
- 保存每道题的提问意图、关联项目、关联技能、抽题理由和追问理由。
- 实现面试总结升级，汇总项目表达、技术深度、知识点掌握、追问表现和后续建议。
- 增加必要的后端测试，覆盖题库语义匹配、面试规划、阶段流转、动态追问和总结生成。

### 前端任务

- 优化主页简历上传流程：用户从主页上传简历并解析成功后，直接进入智能面试配置或创建面试。
- 支持临时上传简历：用户在新建面试页上传的简历可临时解析并用于当前面试。
- 扩展题库创建和编辑页面，支持维护题库描述、适配岗位、技能关键词和领域标签。
- 扩展新建面试页面，支持选择面试流程模式：项目优先或知识优先。
- 在新建面试页面展示大模型推荐的相关题库、推荐理由和可调整选项。
- 实现智能面试配置确认页，展示面试阶段、题量分配、项目追问方向和知识点覆盖范围。
- 升级面试答题页，展示当前阶段、题目来源、关联项目、关联知识点和追问上下文。
- 支持回答后加载下一步决策，区分继续追问、切换知识点、进入下一项目或进入总结。
- 升级面试总结页，展示项目表现、知识点表现、追问表现、优势、短板和后续复习建议。
- 补齐智能面试流程中的加载态、空状态、错误态和大模型失败兜底提示。
- 设计/评审智能面试配置、题库语义信息、动态追问体验和总结页。

### 联调验收

- 用户能在主页上传简历，解析完成后直接进入智能面试流程。
- 用户能在新建面试页临时上传简历，并只用于当前面试。
- 用户能为题库维护描述、适配岗位、技能关键词和领域标签。
- 系统能根据简历、岗位目标和题库描述推荐相关题库，并展示推荐理由。
- 用户能选择“项目优先”或“知识优先”面试流程。
- 项目优先流程能先围绕简历项目提问，再根据项目中暴露的技术点联动知识库题目。
- 知识优先流程能先抽取相关知识库题，再结合回答内容追问简历项目。
- 大模型追问不能编造简历中不存在的项目经历、职责或技术细节。
- 面试过程中每道题都能记录提问意图、抽题理由、关联项目或关联知识点。
- 回答后系统能动态决定继续追问、切换知识点或进入下一阶段。
- 大模型失败时系统能使用规则兜底抽题，并给出可理解提示。
- 完成面试后，用户能看到项目表达、知识掌握、追问表现和复习建议的综合总结。

## 最小可用版本

最小可用版本只要求先跑通：

1. 登录和小组。
2. 题库 CRUD。
3. 飞书导入。
4. AI 抽题入库。
5. 普通面试抽题。

客制化面试、完整评分和总结可以在最小闭环之后继续增强。
