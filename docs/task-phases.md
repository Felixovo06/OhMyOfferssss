# AI 面试抽题网站任务书

## 执行方式

项目按前后端分离推进，但任务阶段按业务闭环划分，适合 vibecoding 快速迭代。

每个阶段只追求一个明确结果：能跑通、能体验、能继续迭代。不要把阶段拆得太碎。

涉及 UI 结构、视觉设计、交互体验、组件状态、响应式布局或 UI 质量评审时，必须使用 `ui-ux-pro-max`。

## 阶段 1：项目骨架与基础能力

目标：前后端项目能跑起来，登录、小组、基础权限和接口规范能打通。

包含：

- 建立 monorepo。
- 创建 `apps/web` 前端项目。
- 创建 `apps/api` 后端项目。
- 创建 `packages/shared` 共享类型和 schema。
- 配置 PostgreSQL、Redis、Prisma。
- 实现登录、退出、当前用户。
- 实现小组创建、邀请、加入。
- 实现基础后台 UI：登录、工作台、小组入口。
- 使用 `ui-ux-pro-max` 设计/评审基础布局、登录页、工作台和小组流程。

验收：

- 前端和后端都能本地启动。
- 前端能调用后端 `/api/v1/auth/me`。
- 用户能登录。
- 用户能创建小组并邀请成员。
- OpenAPI 或接口文档能看到基础接口。

## 阶段 2：题库与飞书导入闭环

目标：能把飞书里的题导入系统，并经过 AI 抽取和人工确认进入题库。

包含：

- 实现题库 CRUD。
- 实现题目 CRUD。
- 支持标签和难度分数。
- 支持个人题库和小组题库。
- 实现飞书 token 获取和缓存。
- 实现飞书文档链接解析。
- 实现飞书 blocks 拉取。
- 实现 blocks 转 normalized text。
- 实现 AI 从飞书内容抽取题目、答案、标签、难度分数。
- 实现导入批次和导入项。
- 实现导入确认和批量入库。
- 实现前端题库列表、题库详情、题目编辑、飞书导入、导入确认。
- 使用 `ui-ux-pro-max` 设计/评审题库管理、导入流程、确认表格、低置信度提示和错误状态。

验收：

- 用户能创建个人题库和小组题库。
- 用户能手动添加题目。
- 用户能输入飞书文档链接并拉取内容。
- AI 能生成待确认题目。
- 用户确认后题目入库。
- 每道题保存标签、难度分数和来源信息。

## 阶段 3：普通面试闭环

目标：用户可以基于题库开始普通面试，大模型负责抽题，用户可以回答并获得反馈。

包含：

- 实现候选题筛选。
- 实现普通面试抽题 prompt。
- 实现大模型抽题。
- 保存抽题理由和整体策略。
- 实现规则兜底抽题。
- 实现面试会话、面试题目、回答记录。
- 实现回答评分、追问和单题反馈。
- 实现会话总结。
- 实现前端普通面试配置、面试答题、反馈、总结。
- 使用 `ui-ux-pro-max` 设计/评审面试配置、答题体验、反馈面板、进度和总结页。

验收：

- 用户能选择题库、标签、难度、题量和目标。
- 大模型能返回题目组合和抽题理由。
- 用户能逐题回答。
- AI 能给出评分、缺失点、参考表达和追问。
- 完成后能看到总结。

## 阶段 4：客制化面试闭环

目标：用户上传 PDF 简历后，可以基于简历和题库进行客制化面试。

包含：

- 实现 PDF 简历上传。
- 实现 PDF 文本解析。
- 判断扫描件或图片型 PDF。
- 实现简历结构化摘要。
- 实现客制化面试抽题 prompt。
- 大模型结合简历摘要和题库候选题抽题。
- 保存与简历相关的追问方向。
- 客制化面试回答评分时参考简历上下文。
- 实现前端简历上传、解析状态、简历摘要、客制化面试配置、答题和总结。
- 使用 `ui-ux-pro-max` 设计/评审简历上传、解析状态、扫描件提示、简历摘要展示和客制化面试体验。

验收：

- 用户能上传文本型 PDF 简历。
- 系统能解析简历文本并生成摘要。
- 扫描件 PDF 有明确提示。
- 大模型抽题理由能体现简历技能、项目或经历。
- AI 追问不能编造简历中不存在的经历。
- 用户能完成一轮客制化面试并看到总结。

## 阶段 5：联调、部署与打磨

目标：把 MVP 打磨到可以给内部用户使用。

包含：

- 前后端接口联调。
- 关键链路端到端测试。
- 错误日志和 requestId。
- 敏感信息检查。
- 环境变量和部署说明。
- 飞书应用配置说明。
- UI/UX 质量检查。
- 加载态、空状态、错误态补齐。
- 移动端基础可用性检查。
- 使用 `ui-ux-pro-max` 做上线前 UI/UX 评审。

验收：

- 登录 -> 创建小组 -> 创建题库 -> 飞书导入 -> AI 抽题入库 -> 普通面试 -> 总结 全链路可跑通。
- 上传简历 -> 解析简历 -> 客制化面试 -> 总结 全链路可跑通。
- 关键错误有可理解提示。
- `.env` 不会被提交。
- README 能指导本地启动和部署。

## 最小可用版本

最小可用版本只要求先跑通：

1. 登录和小组。
2. 题库 CRUD。
3. 飞书导入。
4. AI 抽题入库。
5. 普通面试抽题。

客制化面试、完整评分和总结可以在最小闭环之后继续增强。
